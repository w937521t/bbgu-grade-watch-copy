# Mihomo 节点选择修正设计

## 问题

GitHub Actions 工作流仍按旧订阅配置，只允许名称匹配香港、台湾或日本的节点。更换订阅后，这个地区限制会排除其他可用节点，并可能持续选中无法稳定加载教务系统登录页的节点。

## 设计

- 删除代理提供者中的 HK/TW/JP 地区 `filter`。
- 保留流量、到期、官网、套餐等非代理条目的排除规则。
- 继续使用 Mihomo `url-test`，直接以教务系统登录地址进行健康检查和自动选择。
- 将日志中的“HK/TW/JP节点”改成“代理节点”，避免误导。
- 不修改登录流程、Token 状态、扫码规则、定时规则和加密状态保存。

## 错误处理

如果订阅内所有节点都无法访问教务系统，工作流仍在 12 次检测后明确失败，并输出 Mihomo 诊断日志。

## 验证

- 工作流契约测试确认不存在 HK/TW/JP 地区筛选及旧日志文案。
- 工作流契约测试确认 `url-test` 和教务系统健康检查仍存在。
- 运行完整测试并解析工作流 YAML。
